# 云端截图洞察(Cloud Insight)架构设计

状态(2026-07-19):架构定案,待实施一期。所有关键参数经真机 PoC 实测,复测脚本 `scripts/poc/test_siliconflow_vl.py`。
关联:屏幕记忆(`docs/design/screen-memory.md`,本设计是其隐私红线的显式同意例外)、Chat 覆盖披露(`src-tauri/src/chat/tools.rs`)、AI 日报管线(`src-tauri/src/ai/`)。
需求出处:GitHub issue——云端 API 分析截图以满足总结需求(本地 OCR 质量/性能不满足;客户聊天内容希望自动进日报)。

---

## 1. 定位:与本地 OCR 是两层能力,独立开关

| | 本地 OCR(屏幕文字索引) | 云端 VLM 洞察(本设计) |
|---|---|---|
| 产出 | 逐字文本 → FTS 索引 | 一句话"在干什么" + 关键实体 |
| 服务 | 搜索(逐字命中) | 总结、语义理解 |
| 代价 | 本地 CPU | 按量付费 + 截图出网(显式同意) |

不做二选一:四种组合都是合法形态(都关→标题层兜底;只 OCR→现状;只 VLM→省本地性能;都开→最全)。两条管线消费同一个 frames 登记表,`ocr_state` 与洞察 `state` 各自独立,互不阻塞。

## 2. 隐私闸门(screen-memory 红线的唯一例外通道)

1. **默认关 + 同意门**:开启时明示"所选应用的截图将上传至你配置的服务商",确认落 settings 标记(同 `chatPrivacyAcknowledged` 模式);
2. **UI 永不主动推荐**(项目既有原则);
3. **上传前隐私过滤**:采集端隐私名单本就不截图;上传侧再设黑名单复核。默认在隐私名单内的应用(如微信)需用户自行放行;
4. **出口单一**:仅用户自配的 OpenAI 兼容端点(endpoint/key/model),不预设、不推荐厂商。

## 3. 数据流与落库

```
帧落盘 → 内容粗门 → 应用策略筛帧 → 队列 → 单图单调用 → frame_insights 落一行
```

- **内容粗门**:与上一上传帧比 256×144 灰度差,|Δ|>12 像素占比 ≥10% 才算新画面;同画面至多 10 分钟强制刷新一帧(有界丢失)。注意与存储级 L1 门(0.1%)不是一回事——那是落盘门,砍不动上传量(实测仅 3-6%);
- **帧级落库,不存聚合物**:帧洞察是原始事实;块/日汇总是消费端生成时的临时动作(§6),落库会永久丢粒度。

```sql
CREATE TABLE frame_insights (
  path        TEXT PRIMARY KEY,   -- 对应 frames.path
  ts          TEXT NOT NULL,
  local_date  TEXT NOT NULL,
  app         TEXT, title TEXT,   -- 来自采集数据(事实,非模型输出)
  insight     TEXT,               -- 一句话(模型输出)
  entities    TEXT,               -- 关键实体,逗号分隔(模型输出)
  state       INTEGER NOT NULL DEFAULT 0,  -- 0待 1完 2失败(可重试) 3跳过(门/策略/隐私)
  attempts    INTEGER NOT NULL DEFAULT 0,
  done_at     TEXT                         -- 分析完成时刻;日限额按它计数(回填也占额度)
);
```

## 4. 调用规格

- **单图单调用**:多图一次喂 8B 级 VLM 有实测的注意力稀释(4 帧错丢 2),且输入 token 几乎不省;
- **分辨率**:文字密集内容 1280px 长边是底线(960 起实体退化,768 幻觉);次要应用可降 960 省 41% 输入;
- **输出契约**:一句话 + 实体列表,≤80 字,temperature 0。描述式输出幻觉低危(错了钝化总结,不像逐字转写会往索引塞假事实);
- **时长不靠模型**:活动库有精确时长,帧带时间戳,模型只管内容;
- 参考引擎:硅基流动 Qwen3-VL-8B(PoC 胜出);架构只依赖 OpenAI 兼容 vision chat 格式。

## 5. 应用策略与成本

三档(设置页选择,费用预估实时显示,每日帧数上限超限自动停):

| 档 | 内容 |
|---|---|
| A | 只分析重点应用(用户多选;逐独立帧,1280px) |
| B(默认) | 重点应用逐帧 + 其余每 5 分钟一帧(960px) |
| C | 全应用逐独立帧 1280px |

成本模型:≈ 帧数 × 输入单价(输出 ~50 tok 可忽略)。2026-07 实测口径:¥0.0015/帧(1280px);本机帧量轻/中/重日 219/907/2,064,粗门后存留 58-77%。折合 A 档个位数元/月、B 档十余元/月、C 档几十元/月。时价变动以 PoC 脚本复测为准。

## 6. 消费端

- **日报/周报**:生成 prompt 按小时分组注入帧洞察;token 超限由总结管线 map-reduce,不落库;
- **Chat**:洞察实体接入搜索(独立小节,模式同窗口标题层);证据卡直接挂帧截图;覆盖披露行报告 FTS/洞察/标题三层在线状态,模型措辞跟随覆盖(system_prompt 规则 8 机制);
- **检索回退链**:FTS 逐字层 → 洞察实体层 → 窗口标题层(永远在线)。

## 7. 并发与时序

| 场景 | 并发 | 说明 |
|---|---|---|
| 实时常驻 | 1 | 采集至多 2 帧/分,串行即有 10 倍余量;帧落盘数秒内洞察就绪 |
| 当日积压(功能中途开启/断连恢复) | 16 | 实测吞吐 ~4 帧/s,重度日全量 ≈7 分钟;并发再高服务端已见顶 |
| 历史回填(手动,先显示帧数/费用/时长预估) | 16 | 30 天 ≈2 小时 |

5xx/429 指数退避;`state` 断点续跑;云端失败只标记重试,永不阻塞采集与 OCR。

## 8. 已否决方案(摘要,详证见 PoC 脚本与 git 记录)

| 候选 | 否决原因 |
|---|---|
| 免费 VLM(PaddleOCR-VL / GLM-4V-Flash / DeepSeek-OCR) | 重复死循环、整篇幻觉、输出截断、乱码——忠实度不过门槛 |
| 传统云 OCR API | 按次计费月账单反超 VLM 一个量级;免费额度差需求两个数量级 |
| 抽样(每天/每块取代表帧) | 动态内容(聊天/连续网页)信息损失不可接受 |
| 多图块调用、块级聚合落库 | 见 §4/§3 |

## 9. 前端信息架构(四个语义层,每个配置只有一个家)

| 层 | 回答的问题 | 位置 |
|---|---|---|
| 连接 | 云端资源在哪、凭证是什么 | AI 设置 → 云端 API:「文本模型」(总结/对话用,现状字段)+「视觉模型」(洞察用;默认复用文本端点,只填模型名;测试用本地合成图,不上传真实截图) |
| 分配 | 哪个任务用哪个模型 | AI 设置 → 模型:「任务与模型」三行统一 picker(总结/对话在本地与云端·文本间切;洞察目前仅云端·视觉一项,选项将来自然扩展) |
| 行为 | 洞察开不开、分析什么、花多少 | AI 设置 → 截图洞察(新 tab):开关+同意门(唯一入口)、范围三档+重点应用 chips、每日上限+今日用量、历史回填(帧数确认+进度条+断点续跑) |
| 管线 | 截图之后经过哪几级 | 主设置 → 通用「截图与识别」:屏幕截图 / 常驻文字识别(本地 OCR)/ 截图洞察(云端,状态+跳转,不设第二开关)/ 历史回填(本地 OCR) |

同意门只挂在洞察 tab 的开关上;General 行只显状态,避免双开关绕过同意门。

## 10. 二期候选

逐字级云端转写档(搜索索引质量,成本约一个量级以上)、成本面板、免费模型复测(服务端修复后)。

## 11. 不做什么

不内置/推荐任何厂商;不做己方云服务;不存聚合摘要;媒体观看的闲置豁免另案(显示器防休眠断言)。
