【0.8.4】
- 文字识别（OCR）全面提速：识别改在 GPU 上运行——Windows 支持任意 DX12 显卡（含核显），macOS 改用系统原生识别（无需下载任何组件、运行在神经引擎上）；配合一系列识别管线优化，速度提升约 2~4 倍，并按设备类型（独显/核显）自动调整识别参数。
- 「AI 设置 → 引擎」拆分为两张独立卡片：llama.cpp 引擎（本地总结/对话用）与文字识别（OCR）组件（屏幕记忆用），各自独立下载、卸载。使用云端 API 的用户现在只需约 40MB 的 OCR 组件即可使用全部功能，无需下载数百 MB 的本地引擎。
- 需要文字识别组件的操作会就地引导：开启「常驻 OCR」、点击「立即回填」或对话页提示回填时，组件缺失会先提示下载（约 40MB），完成后自动继续原来的操作。
- 电池供电时「常驻 OCR」自动暂停并释放内存（约 400MB），接回电源后自动恢复并补上积压的截图。
- 使用云端 API 生成总结时，不再要求下载本地 AI 引擎。
- 「对话」页每条回答下方显示本轮 token 用量：↑ 为发送给模型的 token（上行），↓ 为模型生成的 token（下行）；一次提问触发多步查询时自动累计，并随聊天记录保存。

【0.8.4】
- Text recognition (OCR) is much faster: recognition now runs on the GPU — any DX12 graphics card on Windows (integrated included), and macOS switches to the built-in system recognizer (nothing to download, runs on the Neural Engine). Combined with pipeline optimizations, it's roughly 2-4× faster, and recognition parameters auto-tune per device class (discrete vs integrated GPU).
- "AI Settings → Engine" is split into two independent cards: the llama.cpp engine (local summaries/chat) and the text recognition (OCR) component (screen memory), each with its own download and uninstall. Cloud API users now only need the ~40MB OCR component for full functionality — no more multi-hundred-MB engine download.
- Actions that need the OCR component now guide you in place: enabling Resident OCR, clicking Backfill, or accepting the chat page's backfill prompt will offer the ~40MB download first, then continue automatically once it finishes.
- On battery, Resident OCR pauses automatically and frees ~400MB of memory; it resumes and catches up on the backlog when you plug back in.
- Generating summaries with a cloud API no longer requires downloading the local AI engine.
- The Chat page now shows token usage under each answer: ↑ tokens sent to the model (upstream), ↓ tokens generated by the model (downstream). Multi-step queries within one question are summed automatically, and the numbers are kept with your chat history.

【0.8.4】
- 文字認識（OCR）を大幅高速化：認識は GPU 上で実行されます——Windows は任意の DX12 対応 GPU（内蔵 GPU 含む）、macOS はシステム標準の認識（ダウンロード不要、Neural Engine で動作）に切り替わりました。パイプライン最適化と合わせて約 2〜4 倍高速になり、デバイス種別（独立/内蔵 GPU）に応じて認識パラメータを自動調整します。
- 「AI 設定 → エンジン」を 2 枚の独立カードに分割：llama.cpp エンジン（ローカル要約/チャット用）と文字認識（OCR）コンポーネント（画面メモリ用）を、それぞれ個別にダウンロード・アンインストールできます。クラウド API を使う場合、約 40MB の OCR コンポーネントだけで全機能を利用できます。
- OCR コンポーネントが必要な操作はその場で案内：「常駐 OCR」の有効化、「今すぐ取り込み」、チャットページの取り込み案内の際、コンポーネントが無ければ先にダウンロード（約 40MB）を提案し、完了後は自動的に元の操作を続行します。
- バッテリー駆動時は「常駐 OCR」が自動的に一時停止しメモリ（約 400MB）を解放。電源接続で自動再開し、たまった分を取り込みます。
- クラウド API で要約を生成する場合、ローカル AI エンジンのダウンロードは不要になりました。
- 「チャット」ページの各回答の下に token 使用量を表示：↑ はモデルへ送信した token（上り）、↓ はモデルが生成した token（下り）。1 つの質問で複数ステップの照会が走った場合は自動的に合算され、チャット履歴とともに保存されます。

【0.8.4】
- Reconhecimento de texto (OCR) muito mais rápido: o reconhecimento agora roda na GPU — qualquer placa DX12 no Windows (integrada incluída), e o macOS passa a usar o reconhecedor nativo do sistema (nada para baixar, roda no Neural Engine). Com as otimizações do pipeline, ficou cerca de 2-4× mais rápido, com parâmetros ajustados automaticamente por classe de dispositivo (GPU dedicada vs integrada).
- "Configurações de IA → Motor" foi dividido em dois cartões independentes: o motor llama.cpp (resumos/chat locais) e o componente de reconhecimento de texto (OCR) (memória de tela), cada um com download e desinstalação próprios. Usuários de API na nuvem agora só precisam do componente OCR (~40MB) para todas as funções.
- Ações que precisam do componente OCR agora orientam no local: ativar o OCR residente, clicar em Preencher ou aceitar o aviso da página de chat oferece o download (~40MB) primeiro e continua automaticamente ao terminar.
- Na bateria, o OCR residente pausa automaticamente e libera ~400MB de memória; ao reconectar a energia, retoma e processa o acumulado.
- Gerar resumos com API na nuvem não exige mais baixar o motor de IA local.
- A página Chat agora mostra o uso de tokens abaixo de cada resposta: ↑ tokens enviados ao modelo (upstream), ↓ tokens gerados pelo modelo (downstream). Consultas em múltiplas etapas de uma mesma pergunta são somadas automaticamente e salvas com o histórico.
